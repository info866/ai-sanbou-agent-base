# 更新吸収ルール（Update Absorption Rules）

> **正本**: 11.フェーズ5要件定義書.md  
> **最終更新**: 2026-04-13

---

## この文書の役割

新しい能力が出現したとき、既存能力の状態が変わったとき、AI参謀の判断をどう更新するかのルールを定義する。

---

## 更新吸収の2つのトリガー

### トリガー1｜新機能発見

新しいツール・API・OSS・公式機能を発見した場合。

**判定フロー（Phase 4の03_new_tool_intake_rules.mdを継承）:**

```
新機能を発見
  │
  ├─ 1. 直接影響判定: 現在の作業/依頼に直接効くか?
  │    └─ NO → 記録だけして保留（P4低優先に追加）
  │
  ├─ 2. 既存比較: 既存の採用候補（D1/D2）より改善できるか?
  │    └─ NO → 記録だけして保留
  │
  ├─ 3. セットアップ時間: 導入にどれだけかかるか?
  │    ├─ <30分 → 即時採用
  │    ├─ 30〜120分 → 試験採用
  │    └─ >120分 → 次回以降に検討
  │
  └─ 4. 判定結果を記録
       ├─ 採用 → D1/D2に追加、P1〜P3に配置
       ├─ 試験 → D2に追加、試験計画を作成
       └─ 保留 → D3に追加、解消条件を記録
```

**記録すべき情報:**
- 候補名、発見日、情報源URL
- 何に効くか
- 既存候補との比較結果
- 採用/試験/保留の判定と理由

---

### トリガー2｜既存判断の陳腐化検知

既存の採用判断が古くなった可能性がある場合。

**検知条件:**

| 条件 | 例 | 確認方法 |
|------|-----|---------|
| 公式仕様変更 | APIパラメータ追加、破壊的変更 | リリースノート、公式ドキュメント |
| 提供状態変更 | beta→GA、GA→deprecated | リリースノート |
| 有力OSS更新 | メジャーバージョン、方針変更 | GitHubリリース |
| より良い代替候補出現 | 新OSSがstars急増、公式が新機能リリース | GitHub検索、公式発表 |

**陳腐化検知後のフロー:**

```
陳腐化を検知
  │
  ├─ 1. 影響範囲特定: どの採用判断・どの能力選定に影響するか?
  │
  ├─ 2. 差分確認: 何が変わったのか? 変更の程度は?
  │    ├─ 軽微（パラメータ追加等） → 記録のみ
  │    └─ 重大（仕様変更、廃止等） → 再評価へ
  │
  ├─ 3. 再評価: 変更後の状態でPhase 2の評価基準（E1〜E10）を再適用
  │
  └─ 4. 判断更新
       ├─ 採用状態の変更（D1→D3、D3→D1等）
       ├─ 優先順位の変更（P1→P2、P4→P2等）
       └─ 実行フローへの影響反映
```

---

## 更新後の反映先

更新結果は以下に反映する。

| 更新内容 | 反映先 |
|---------|--------|
| 新候補の追加 | Phase 2 候補カタログ（概念的。実運用ではメモリまたは知識基盤に記録） |
| 採用状態の変更 | 能力選定ルール（03_capability_selection.md）の候補プールに影響 |
| 優先順位の変更 | 能力選定ルール（03_capability_selection.md）の選定順序に影響 |
| 実行フローの変更 | 実行橋渡しルール（05_execution_handoff.md）の作業順に影響 |
| 信頼区分の変更 | 再確認判断（03_capability_selection.md）の判定に影響 |

---

## 定期的な更新確認

トリガーによる受動的な更新だけでなく、以下の能動的確認も行う。

| 確認対象 | 頻度 | 確認方法 |
|---------|------|---------|
| P1最優先候補（6件）の公式情報 | フェーズ移行時 | 公式ドキュメント・リリースノート |
| T4再確認前提候補の状態 | 月1回 | リリースノート・GitHub |
| D5監視継続候補の更新 | 3ヶ月毎 | GitHubリポジトリ確認 |
| D3保留候補の解消条件成立 | 比較評価実施前 | 各候補の解消条件を個別確認 |

---

## 更新吸収の出力形式

```
更新種別: [新機能発見 / 陳腐化検知]
対象: [候補名またはitem_id]
変更内容: [何が変わったか]
影響範囲: [どの判断に影響するか]
判定結果: [採用状態・優先順位の変更内容]
実行フローへの影響: [あり/なし。ありの場合は具体的内容]
```
