# フェーズ完了報告スキル

このスキルは、フェーズの各グループの完了報告書を定型フォーマットで生成します。

## 実行手順

1. 現在のフェーズの作業契約書を読む
   - `phase2_decision_foundation/00_scope_and_execution_contract.md`

2. 指定されたグループの成果物を確認する
   - `git log --oneline -10` で直近の作業を確認
   - `git diff --stat HEAD~1` で変更量を確認

3. 以下のフォーマットで報告書を生成する

## 出力フォーマット

```
## 完了報告: フェーズX グループY

### 1. 実施概要
- 固定したこと: ...
- あえてまだやっていないこと: ...

### 2. 作成・更新ファイル一覧
| ファイル | 役割 | 操作 |
|---------|------|------|
| ... | ... | 新規/更新 |

### 3. 成果物の要点
（各成果物の要点を3行以内で）

### 4. 逸脱防止確認
- [ ] 正本文書を変更していない
- [ ] rename/move/deleteをしていない
- [ ] 当グループの責務範囲内に留まっている
- [ ] 有料API/SaaSを使用していない

### 5. GitHub確認情報
- リポジトリ: https://github.com/info866/ai-sanbou-agent-base
- コミットハッシュ: ...
- 差分確認: git diff [前回ハッシュ]..[今回ハッシュ]
- レビュー重点箇所: ...

### 6. 後続グループへの引き継ぎ
- 後続が読むべきファイル: ...
- 後続が注意すべき点: ...
```
