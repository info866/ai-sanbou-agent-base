# 検索要件定義（Search Requirements）

> **ステップ対応**: フェーズ3作業指示書 ステップ5  
> **最終更新**: 2026-04-13

---

## この文書の役割

後からどんな問いで何を引くかを先に固定する。問いの種類ごとに、入力条件・参照する知識層・期待する出力・鮮度確認の要否を定義する。

---

## SR-01｜既知候補検索

**目的**: 特定の候補について詳細情報を引く。

| 項目 | 内容 |
|------|------|
| 入力条件 | item_id（F-001等）またはitem_name（部分一致） |
| 参照する知識層 | KL-1（メタデータ） → KL-2（判断記録） → KL-3（文書知識） |
| 期待する出力 | 候補の全構造化項目 + 判断理由 + 関連文書知識 |
| 鮮度確認 | recheck_required_flag = true の場合、出力時に「再確認推奨」を付記する |

**代表的な問い**:
- 「F-032 promptfooの評価結果を見せろ」
- 「LiteLLMの採用状態と理由は何か」
- 「MCP Connectorの一次情報URLはどこか」

---

## SR-02｜用途別検索

**目的**: ある作業目的に効く候補を探す。

| 項目 | 内容 |
|------|------|
| 入力条件 | 作業適用先コード（W01〜W08）、機能層コード（L01〜L08）、または自然言語の用途記述 |
| 参照する知識層 | KL-1（layer_category, primary_use_casesでフィルタ） → KL-2（関連判断の補足） |
| 期待する出力 | 条件に合致する候補一覧（ID, 名前, 採用状態, 優先順位付き） |
| 鮮度確認 | 出力候補のうちKL-4に属するものは再確認要否を明示する |

**代表的な問い**:
- 「MCP評価に使える基盤は何か」
- 「orchestration層で採用候補になっているものは何か」
- 「実行層（L04）のGA済み公式機能を一覧してくれ」

---

## SR-03｜判断根拠検索

**目的**: ある候補の採用/不採用/保留の理由を引く。

| 項目 | 内容 |
|------|------|
| 入力条件 | item_id + 判断種別（decision_state / build_vs_buy_state） |
| 参照する知識層 | KL-2（reason_summary, compared_with, unresolved_points） |
| 期待する出力 | 判断理由の全文、比較対象、未解決論点、再確認条件 |
| 鮮度確認 | 判断根拠が鮮度依存情報に基づいている場合はその旨を明示する |

**代表的な問い**:
- 「なぜmcp-evalは不採用になったのか」
- 「CrewAIが保留されている理由と解消条件は何か」
- 「Subagentsを全面流用する理由は何か」

---

## SR-04｜比較結果検索

**目的**: 同役割候補の比較結果を引く。

| 項目 | 内容 |
|------|------|
| 入力条件 | 比較グループID（CG-01〜CG-06）または候補のcompared_with |
| 参照する知識層 | KL-2（比較グループ別評価結果） → KL-1（各候補のスコア） → KL-3（詳細比較メモ） |
| 期待する出力 | 比較グループ内の全候補のスコア比較 + 判断結果 + 差異の要約 |
| 鮮度確認 | 比較対象にKL-4知識が含まれる場合、比較結果自体の有効性を注記する |

**代表的な問い**:
- 「router候補（CG-01）の比較結果を見せろ」
- 「evaluation基盤（CG-03）で最も評価が高い候補はどれか」
- 「LangGraphとAgent SDKの比較状況は」

---

## SR-05｜再確認対象検索

**目的**: 現時点で再確認が必要な候補を抽出する。

| 項目 | 内容 |
|------|------|
| 入力条件 | recheck_required_flag = true、またはlast_checked_atが閾値より古い |
| 参照する知識層 | KL-4（再確認要知識層） → KL-1（対応する構造化項目） |
| 期待する出力 | 再確認が必要な候補一覧（ID, 名前, 再確認理由, 最終確認日, 再確認条件） |
| 鮮度確認 | この検索自体が鮮度管理のための検索なので、常に最新状態で実行する |

**代表的な問い**:
- 「いま再確認が必要な候補はどれか」
- 「最終確認日が3ヶ月以上前の候補を一覧しろ」
- 「beta状態の候補でGA移行を監視すべきものは」

---

## SR-06｜未解決論点検索

**目的**: まだ決着していない論点や追加調査が必要な候補を引く。

| 項目 | 内容 |
|------|------|
| 入力条件 | unresolved_points が空でない候補、またはdecision_state = D3（保留） |
| 参照する知識層 | KL-2（未解決論点の記述） → KL-4（再調査条件） → KL-1（候補の現状態） |
| 期待する出力 | 未解決論点一覧（論点内容, 関連候補, 解消に必要な情報, 解消条件） |
| 鮮度確認 | 未解決論点の中に鮮度依存の解消条件がある場合、現状を確認してから回答する |

**代表的な問い**:
- 「まだ決着していない比較論点は何か」
- 「追加調査が必要な候補はどれか」
- 「保留中の候補の解消条件が成立しているものはあるか」

---

## 検索要件のまとめ

| 検索ID | 検索名 | 主参照層 | 鮮度確認 |
|--------|--------|---------|---------|
| SR-01 | 既知候補検索 | KL-1 → KL-2 → KL-3 | recheck_flagに従う |
| SR-02 | 用途別検索 | KL-1 → KL-2 | KL-4所属候補のみ |
| SR-03 | 判断根拠検索 | KL-2 | 根拠の鮮度依存性を注記 |
| SR-04 | 比較結果検索 | KL-2 → KL-1 → KL-3 | 比較対象のKL-4注記 |
| SR-05 | 再確認対象検索 | KL-4 → KL-1 | 常に最新状態で実行 |
| SR-06 | 未解決論点検索 | KL-2 → KL-4 → KL-1 | 解消条件の鮮度確認 |
