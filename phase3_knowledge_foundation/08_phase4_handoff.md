## パート2｜監督AIが参照すべき知識

フェーズ5のAI参謀（監督AI）が知識基盤から参照すべき知識の設計方針。

### 2-1 提案生成のための知識

監督AIが「今回は何を使うべきか」を提案するために、以下を参照する。

- **候補カタログの構造化メタデータ（KL-1）**: 全候補の採用状態・優先順位・機能層を横断検索
- **判断記録（KL-2）**: なぜその候補を推奨するかの根拠を提供
- **文書知識（KL-3）**: 候補の詳細な使い方・設定方法をRAGで検索

### 2-2 安全判断のための知識

監督AIが「この候補を使って安全か」を判断するために、以下を参照する。

- **信頼区分（T1〜T4）**: T4の知識に基づく候補は再確認必須
- **再確認フラグ**: recheck_required_flag = true の候補は使用前に確認
- **安全性スコア（E7）**: E7が低い候補のリスクを把握
- **保留理由**: D3（保留）候補を誤って使用しないように

### 2-3 学習・改善のための知識

監督AIが自身の判断を改善するために、以下を蓄積・参照する。

- **過去の判断記録（KL-2）**: 過去にどう判断したかの履歴
- **未解決論点（KL-2）**: まだ結論が出ていない問題の認識
- **更新追跡ルール**: 何をいつ再確認すべきかの運用知識

## パート3｜知識基盤と実行基盤の接続点

### 3-1 接続インターフェースの設計方針

フェーズ4で実行基盤を構築する際、知識基盤との接続は以下の方針で設計する。

**接続方式の候補**（フェーズ4で確定）:
1. **ファイルベース参照**: phase1/phase2/phase3の成果物ファイルを直接Readする。最もシンプル
2. **構造化DB**: 知識項目スキーマ（07_knowledge_schema.md）に基づきSQLiteまたはJSONDBに格納。フィルタ・ソート検索を高速化
3. **RAG + 構造化DBのハイブリッド**: KL-3をRAGで、KL-1をDBで管理。最も柔軟だが実装コスト高

**フェーズ3時点の推奨**: 方式1（ファイルベース参照）で開始し、候補数や検索頻度の増加に応じて方式2→3へ段階的に移行する。

### 3-2 知識基盤の更新経路

実行フェーズで知識が更新される場面と、その更新方法。

| 更新場面 | 更新対象 | 更新方法 |
|---------|---------|---------|
| 新候補の発見 | KL-1（メタデータ）, KL-3（文書知識） | phase1/02に追加し、知識スキーマに従い拡張項目を付与 |
| 既存候補の再評価 | KL-1（スコア更新）, KL-2（判断記録更新） | phase2/04の評価記録を更新し、last_verified_atを更新 |
| beta→GA移行 | KL-1（beta_or_ga）, KL-4→KL-1（信頼区分昇格） | 公式一次情報で確認後、trust_levelをT4→T1へ変更 |
| 不採用→再検討 | KL-1（decision_state）, KL-2（理由更新） | 再検討条件の成立を確認し、D4→D3またはD2へ変更 |
| 公式仕様の変更 | KL-3（文書知識更新）, KL-4（鮮度情報更新） | 新しい仕様を反映し、last_verified_atを更新 |

## パート6｜未解決論点（フェーズ4以降で解消）

| # | 論点 | 理由 | フェーズ4での扱い |
|---|------|------|-----------------|
| 1 | 知識基盤の接続方式（ファイルベース vs 構造化DB vs ハイブリッド） | フェーズ3は設計フェーズのため実装方式は確定しない | フェーズ4開始時に候補数・検索頻度に基づき確定する |
| 2 | RAGの具体的な技術選定（ベクトルDB、エンベディングモデル） | 本フェーズでは適用範囲の定義のみ | フェーズ4でKL-3の資産量が十分になった時点で選定する |
| 3 | Langfuse vs Arize Phoenixのオブザーバビリティ基盤選定 | フェーズ2でD3（保留）のまま | フェーズ4でエージェント実装と同時に試用比較する |
| 4 | Memory Tool API vs auto memoryの役割分担 | 両方が並行利用されている状態 | フェーズ4の知識永続化設計で明確に分離する |
| 5 | Agent SDK単体でのorchestration能力の限界値 | 実装なしに判断できない | フェーズ4で実際にAgent SDKを使ったワークフローを構築し限界を確認する |
