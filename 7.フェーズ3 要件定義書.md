# 【フェーズ3要件定義書 / タスク名】
お前の今回の任務は、**フェーズ1で整備した情報基盤と、フェーズ2で整備した判断基盤を入力として受け取り、それらを「後から検索できる」「必要時に再利用できる」「根拠付きで取り出せる」「古い知識を鵜呑みにしない」知識活用基盤へ変換するための要件を固定すること**である。

この文書は、プロジェクト全体のうち **フェーズ3（知識活用基盤フェーズ）** の要件定義書である。  
ここで定義するのは、**どの情報を知識資産として残すか、どのように検索・参照・再利用するか、どの条件でRAG的活用を行うか、どの条件で外部再調査を必須にするか、後続フェーズで実行基盤やAI参謀にどう接続するか** である。  
この文書の段階で、完成版AI参謀の本番実装、本番オーケストレーターの構築、全面自動運用の完成をやろうとするな。  
このフェーズの役割は、**判断済み資産を「使える知識」に変えること** にある。

---

# 【このフェーズの位置づけ】
全体の大分類フェーズは以下である。

1. **フェーズ1：情報基盤**
2. **フェーズ2：判断基盤**
3. **フェーズ3：知識活用基盤**
4. **フェーズ4：実行基盤**
5. **フェーズ5：AI参謀完成**

今回扱うのは、このうち **フェーズ3のみ** である。  
フェーズ3の役割は、フェーズ1で収集した情報と、フェーズ2で評価・比較・優先順位付けした判断資産を、**検索可能・再利用可能・更新追跡可能・根拠参照可能な知識基盤** に変換することにある。

---

# 【最重要目的】
このフェーズの最重要目的は以下である。

1. **散在している情報・評価・判断を、後から引ける知識資産へ変換すること**
2. **必要なときに、必要な情報だけを、根拠付きで取り出せること**
3. **構造化データと非構造文書を、目的別に使い分けられること**
4. **高速に変化する情報について、保存知識だけで断定せず、再確認すべき条件を持つこと**
5. **RAGを含む知識検索・参照の土台を定義し、後続フェーズで実務投入できる状態にすること**
6. **フェーズ4以降の実行基盤と、最終的なAI参謀エージェントが知識を再利用できるようにすること**

---

# 【背景】
フェーズ1では「何があるか」を集め、整理し、状態管理できるようにした。  
フェーズ2では「何を使うべきか」を比較し、評価し、判断できるようにした。  
しかし、情報と判断が存在していても、それを **必要なときに即座に引き出せない** なら、実務ではほとんど価値が出ない。

このフェーズで解決すべき問題は、単なる「検索の不便さ」ではない。  
本質は、**収集済み情報・判断済み情報・比較結果・公式根拠・OSS候補・保留理由・再評価条件が、後から実際の作業に結びつく形で再利用されていないこと** にある。

さらに、このプロジェクトが扱う対象は変化が速い。  
Claude Code 周辺、Anthropic の新機能、MCP ecosystem、公開OSS、既存エージェントは、継続的に更新される。  
したがって、このフェーズでは「一度保存した知識を絶対視する」構造にしてはならない。  
必要なのは、**保存知識を高速活用しつつ、古い可能性があるものは再検証へ回せる知識基盤** である。

# 【このフェーズで達成すべき到達状態】
フェーズ3完了時点で、最低限以下の状態に到達していなければならない。

## 到達状態G1｜知識資産の分類完了
フェーズ1・フェーズ2で作られた情報と判断が、知識資産として分類されていること。

## 到達状態G2｜検索可能化
後から、用途・役割・評価結果・状態・優先順位・情報源・更新時期などで検索可能になっていること。

## 到達状態G3｜再利用可能化
一度調べたこと・一度比較したこと・一度下した判断を、後から再利用できること。

## 到達状態G4｜根拠追跡可能化
どの知識が、どの情報源・どの比較・どの評価・どの判断に基づいているか追跡できること。

## 到達状態G5｜再確認条件の保持
鮮度が重要な知識、変更されうる知識、公式再確認が必要な知識について、再確認条件が保持されていること。

## 到達状態G6｜RAG活用前提の整理
RAG的検索・参照・要約に載せる資産と、載せずに構造化DBやメタデータ参照で扱う資産が切り分けられていること。

## 到達状態G7｜後続接続性
フェーズ4の実行基盤やフェーズ5のAI参謀が、この知識基盤を参照して意思決定・作業提案・能力選定を行える状態になっていること。

- 情報源

- 仕様変更されやすいもの
- ベータ / GA 状態が変わるもの
- 更新頻度が高いOSS
- リリースノート依存の知識

# 【このフェーズで定義すべき検索の役割】
知識活用基盤は、少なくとも以下の種類の検索に対応できる前提を持て。

## 検索Q1｜既知候補検索
名前・種別・役割・状態から、特定候補を引く検索。

## 検索Q2｜比較結果検索
同役割候補の比較結果や、過去に何と比較したかを引く検索。

## 検索Q3｜判断根拠検索
採用 / 保留 / 不採用の理由を引く検索。

## 検索Q4｜用途別検索
「この作業に効くものは何か」を探す検索。

## 検索Q5｜再確認対象検索
「今この知識を再確認すべきか」を判定するための検索。

## 検索Q6｜公式根拠検索
最終的な裏取りや引用元として、公式情報を優先的に引く検索。

## 検索Q7｜未解決論点検索
まだ決着のついていない論点や比較不足領域を引く検索。

# 【このフェーズで定義すべき知識活用フロー】
知識活用は、最低限以下の流れで行える前提を持て。

## フローF1｜問い合わせの分類
ユーザーまたは後続エージェントの問い合わせが、
- 既知候補参照
- 比較結果参照
- 判断根拠参照
- 新規探索要求
- 再確認要求
のどれかを判定する。

## フローF2｜知識源の選択
問い合わせ内容に応じて、
- 構造化メタデータ
- 判断記録
- 文書知識
- 外部再調査
のどれを優先するか決める。

## フローF3｜検索実行
必要な知識を検索し、関連度の高いものを抽出する。

## フローF4｜根拠紐付け
検索結果に対して、元情報源・元比較・元判断を紐付ける。

## フローF5｜鮮度判定
その知識が古い可能性があるか、公式再確認が必要かを判定する。

## フローF6｜出力形成
知識を要約し、必要なら
- 確定情報
- 判断済み情報
- 再確認必要情報
- 未解決情報
に分けて返せるようにする。

# 【このフェーズで定義すべきRAGの位置づけ】
このフェーズでは、RAGを魔法の万能解決策として扱うな。  
RAGは **知識活用の一部** であって、全体そのものではない。

## 要件R1｜RAG対象の切り分け
RAGに載せるべきものと、載せるべきでないものを分けること。

### RAG向き
- 長文ドキュメント
- README
- 比較メモ
- 評価メモ
- 公式解説文
- 実装パターン説明

### RAG非向きまたは構造化優先
- 状態管理
- 優先順位
- 採用状態
- 再評価条件
- 確定メタデータ
- 更新日時
- 一意ID管理

## 要件R2｜RAGは補助である
RAGから得た知識だけで断定しない。  
必要に応じて構造化メタデータや公式再確認と組み合わせること。

## 要件R3｜鮮度重要情報の再確認
最新性が重要な話題は、RAGに入っているからといって正しいと見なさない。  
必要条件に応じて、公式情報やGitHubへ再調査をかける前提を持つこと。

## 要件R4｜引用可能性
後から「なぜその知識を出したのか」が追跡できるように、出典や元情報への接続性を保持すること。

---

# 【このフェーズで定義すべき再確認ルール】
保存知識の再利用は、以下のルールを前提としろ。

## 再確認C1｜鮮度依存再確認
以下は原則として再確認対象。
- release notes に依存する知識
- ベータ / GA 状態が変わりうる知識
- 価格・モデル・仕様・制限に関わる知識
- 活発に更新されるOSSの機能知識

## 再確認C2｜信頼不足再確認
知識の根拠が弱い、または比較対象が少ない場合は再確認対象とする。

## 再確認C3｜重要判断前再確認
採用判断、実装投入判断、監督ルール化に進む前には、必要に応じて公式・GitHub再確認を行う。

## 再確認C4｜RAG単独回答禁止条件
以下では、RAG検索結果だけで断定しない。
- 最新機能の有無
- 仕様変更の有無
- 現在の正式名称 / 提供状態
- 安全性や権限条件
- 既存OSSの最新活性度

# 【このフェーズで定義すべき知識の信頼区分】
知識は少なくとも以下の区分で扱え。

## 信頼T1｜一次確定知識
公式一次情報または明確な構造化記録に基づく知識。

## 信頼T2｜判断済み知識
比較・評価・採用判断を経た内部知識。

## 信頼T3｜参考知識
有用だが、断定には使わず、参考扱いに留める知識。

## 信頼T4｜再確認前提知識
存在は把握しているが、利用前に再確認が必要な知識。

# 【このフェーズで必ず守るべき前提】
## 前提A1｜保存知識 = 絶対真実ではない
保存した知識を、そのまま現在の真実だと決めつけるな。

## 前提A2｜構造化と文書化を混ぜるな
何でもRAGへ入れるな。  
構造化で扱うべきものと、文書検索で扱うべきものを分けろ。

## 前提A3｜再利用を最優先する
一度作った比較・判断・要約を、後からもう一度使えるようにすることを優先しろ。

## 前提A4｜根拠のない知識を出さない
出典・比較元・判断元が追えない知識は、信頼レベルを下げて扱え。

## 前提A5｜公式再確認を捨てない
変化しうる知識については、外部再調査を前提にする。

# 【絶対ルール】
- RAGを万能解決策として扱うな
- 保存知識だけで最新情報を断定するな
- 構造化すべきものを全部文書検索に流すな
- 逆に、長文知識を全部メタデータ化しようとするな
- 根拠追跡できない知識を高信頼扱いするな
- 再確認条件を持たない知識運用にするな
- フェーズ4の実装を前倒しするな
- フェーズ5のAI参謀完成を前倒しするな
- 「検索できる」だけで満足するな
- 「再利用できる」「根拠が追える」「古い知識を見抜ける」状態まで意識しろ
