# 【追加5層 統合作業指示書 / タスク名】
お前の任務は、**既存の頭脳を使いながら、追加5層を優先順位順に構築し、完成した層はその瞬間から後続作業へ最大限投入すること** である。

---

# 【今回の目的】
この作業でやることは次の5つだけである。

1. **実行制御層を作る**
2. **自動品質ゲート層を作る**
3. **接続・認証・権限自動化層を作る**
4. **常時監視・自動同期層を作る**
5. **継続評価・改善層を作る**

---

# 【作業原則】
## 1. 順番を守る
実装順は固定する。  
**1 → 2 → 3 → 4 → 5** 以外で進めるな。

## 2. 完成した層は即使う
各層は完成後ただちに後続層の実装へ投入する。  
保留しない。

## 3. 頭脳を前提にする
既存の依頼分類、能力選定、モデル選定、再確認判断、実行橋渡しは再実装しない。  
追加5層はそれを強化するために作る。

## 4. 手動依存を減らす
ユーザーが slash command や個別機能を覚えて操作する前提にするな。  
実行制御層が裏で扱うように組む。

## 5. 長文化しない
作業に必要なことだけ扱う。  
説明を増やすな。

---

# 【入力】
必ず以下を使え。

- 既存の頭脳
- 既存の知識基盤
- 既存の実行基盤
- GitHub 上のコードと履歴
- 利用可能な公式機能 / MCP / API / OSS
- 実装中に新しく見つかる有効機能

---

# 【標準作業フロー】
以下の順で進めろ。

## STEP 1｜対象固定
最初にこれだけ決める。
- 今回作る層
- その層の目的
- 完了条件
- 今回やらないこと

## STEP 2｜既存基盤確認
新しい層を作る前に、既存の頭脳と既存基盤のどこを使うか確認する。  
重複実装は禁止。

## STEP 3｜最小構成で実装
その層の中核機能だけを先に作る。  
周辺機能を先に増やすな。

## STEP 4｜検証
最低限、以下を確認する。
- 目的を満たすか
- 既存基盤とつながるか
- 後続層の実装で使えるか
- 壊していないか

## STEP 5｜即投入
完成した層を、次の層の実装にすぐ使う。  
これを標準動作にする。

---

# 【層ごとの作業】
## 1. 実行制御層
### やること
- 頭脳の判断結果を受ける
- slash command / subagent / hook / 各種ツールを自動選択する
- 実行順序を固定する
- ユーザー手動操作を減らす

### 完了条件
- 機能の自動発火が可能
- 実行順序が明確
- 後続層の実装で使える

---

## 2. 自動品質ゲート層
### やること
- 実行制御層を使って自動検証を差し込む
- テスト、差分確認、危険変更検知、失敗停止を組み込む
- 品質を通過しない処理を先へ進めない

### 完了条件
- 主要作業に品質ゲートが入る
- 危険変更を自動検知できる
- 実行制御層経由で動く

---

## 3. 接続・認証・権限自動化層
### やること
- 実行制御層と品質ゲート層を使って接続処理を自動化する
- MCP / API / GitHub / 公式機能を利用可能状態へ持っていく
- 権限不足や未接続で止まりにくい構造にする

### 完了条件
- 必要機能が使える状態まで整う
- 接続不足が早期検知される
- 後続層から自動利用できる

---

## 4. 常時監視・自動同期層
### やること
- 前段3層を使って監視対象を巡回する
- 更新、新機能、破壊的変更を検出する
- 重要更新だけを抽出する
- 判断基盤と知識基盤へ更新対象を渡す

### 完了条件
- 継続監視できる
- 重要変更だけを上げられる
- 再評価へ接続できる

---

## 5. 継続評価・改善層
### やること
- 前段4層の実行結果を計測する
- モデル選定、能力選定、ツール選定、実行フローを見直す
- 改善結果を次回以降へ反映する

### 完了条件
- 実行結果を測定できる
- 改善点を特定できる
- 運用へ反映できる

---

# 【検証ルール】
各層の完了前に必ず確認する。

- 既存の頭脳と重複していないか
- 後続層で再利用できるか
- 自動化の方向に進んでいるか
- 手動操作を増やしていないか
- 効率と品質の両方を上げているか

---

# 【禁止事項】
- 順番を入れ替えること
- 完成した層を使わず次へ進むこと
- 既存の頭脳を作り直すこと
- 手動操作前提に戻ること
- 長い設計説明を増やすこと
- 周辺機能から先に作ること

---

3. 既存の頭脳を前提にしている
4. 手動依存が減っている
5. 効率化と品質向上が累積している
6. 次回以降もこの構造で自己強化できる
