# 【フェーズ5要件定義書 / AI参謀統合】

フェーズ1〜4で整備した情報基盤・判断基盤・知識活用基盤・実行基盤を統合し、**実際の業務遂行をリードするAI参謀エージェント** を実装・運用する。

---

## 【このフェーズの目的】

フェーズ5の目的は、次の3つだけである。

1. **実行基盤（フェーズ4）をAI参謀が理解・使用できる形に統合する**
2. **新しい要件・課題が発生したときに、AI参謀が自律的に判断・実行・記録できるエージェント を実装する**
3. **フェーズ4で整備した7つの実行ルール・フロー・判定ロジックをAI参謀エージェントに組み込み、継続的に運用可能な状態にする**

---

## 【このフェーズの位置づけ】

- フェーズ1：情報を集める
- フェーズ2：何を使うか決める
- フェーズ3：後から引ける知識にする
- フェーズ4：それらを使って実際に最速・高品質で作業する
- **フェーズ5：AI参謀として統合し、継続的に実行・改善する**

## 【最重要原則】

### 原則1｜フェーズ4実行基盤を正本とする
フェーズ4で整備した「実行基盤」は不可侵。AI参謀はこの基盤を正本として参照し、実行する。
基盤の修正が必要な場合は、フェーズ4へ戻って修正してから、Phase 5での使用を再開する。

### 原則2｜AI参謀は「判定と実行」を自動化する
AI参謀がすべき仕事は、「判定」と「実行」と「記録」だけ。
人間の確認を待つ判断は（よほどの例外以外）しない。

### 原則3｜新しい要件・課題は AI参謀が判定する
新しい要件・課題が発生したら、AI参謀が以下を自動実行：
1. 実行フロー判定 → 現在位置と次ステップを判定
2. ツール選定 → ツール最大活用方針から最適ツール選定
3. 作業単位化 → 作業単位定義から粒度を判定
4. 実行 → 該当フローで実行
5. 検証 → 4階層自己検証
6. 記録 → GitHub へ記録・push

### 原則4｜フェーズ4実行基盤の質を守る
AI参謀の判定に失敗・誤判定があった場合は、フェーズ4実行基盤の「品質保証ルール」に戻って問題を分析し、
基盤自体の改善が必要か、AI参謀の判定ロジックの改善が必要かを判定する。

### 原則5｜継続的に改善する
AI参謀が実行を積み重ねる中で、
- 新しい有効ツールが発見される → 03_new_tool_intake_rules.md で評価・採用
- 新しい有効なフローが見つかる → 04_work_unit_definitions.md で吸収
- 新しい品質基準が必要 → 05_quality_assurance_rules.md で組み込み

このような改善はフェーズ5内で実行。基盤自体の修正はフェーズ4へ戻す。

   - 入力：新しい要件・課題

   - Phase 4 実行基盤を自動統合

## 【このフェーズで定義すべき内容】

### 1. AI参謀エージェントの実装仕様

- **初期化**: Phase 4 実行基盤（7つのファイル）を読み込み、内部状態として保持
- **主実行ループ**: 新規要件受信 → 自動判定 → 自動実行 → 自動検証 → 自動記録
- **判定ロジック**: 5つの自動判定関数（Phase 4 HandOff Memoで提供済み）
- **エラーハンドリング**: 異常検出時は品質保証ルール準拠で対応
- **ログ・トレーサビリティ**: すべての判定・実行をGitHubに記録

### 2. AI参謀の使用するツール・リソース

- Claude API（複数モデル選択）
- MCP（Model Context Protocol）
- Subagents（専門家エージェント）
- Hooks（イベント駆動）
- GitHub API（差分・コミット・PR操作）
- 計算資源・API額のコスト管理

### 3. 監視・ログ・改善サイクル

- AI参謀の実行履歴（何を判定したか、なぜか）
- エラー・異常検出ログ
- 継続的改善の仕組み（新規ツール・フロー発見→評価→採用）
- 定期的な効果測定（精度・速度・品質向上）

### 4. フェーズ1〜4との連携インターフェース

- Phase 1 情報参照 API
- Phase 2 採用状態参照 API
- Phase 3 知識活用ルール実装
- Phase 4 実行基盤の読み込み・参照・実行

## 【このフェーズでまだやらないこと】

以下はフェーズ5ではまだやらない。

- 完全自律運用の永遠保証
- 人間確認ゼロの全自動化（リスク高い判定は人間確認を入れる）
- フェーズ5以降の完成物設計
- 全ツール・全機能の完全網羅
- AI参謀の最終パーソナライズ（別フェーズ）

## 【絶対ルール】

- フェーズ4実行基盤を改変するな（参照だけを許す）
- AI参謀の判定が基盤より優先することはない
- すべての実行をGitHubに記録しない者は、運用対象外とする
- エラーを隠蔽するな（品質保証ルール準拠）
- 人間確認が必要な判定は、その旨を明示して待つ
